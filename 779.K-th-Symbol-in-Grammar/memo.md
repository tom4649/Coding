## 問題文から直接 sol3 にたどり着く

結論: たどり着ける。むしろ「前半=前の行、後半=前の行のflip」（sol1）の特徴づけより、置換規則「0→01, 1→10」をそのまま読んだ方が popcount 解にすぐ到達する。

### 置換規則を「親→子」のインデックス対応で読む

```
0 → 0 | 1     (右側の 1 は「0 を flip」)
1 → 1 | 0     (右側の 0 は「1 を flip」)
```

つまり、行 n の各位置 `j` (0-indexed) について

- `j` が偶数なら、行 n-1 の位置 `j/2` の値そのまま
- `j` が奇数なら、行 n-1 の位置 `j//2` の値の **flip 1 回**

`j = k - 1` を 2 で割り続けて 0 に到達するまでの過程で、flip が起きるのは **`j` を 2 進表記したときに 1 が立っていた回数**だけ。

行 1 の値は `0` なので、

\[
\text{答え} = \text{popcount}(k - 1) \bmod 2
\]

→ `(k - 1).bit_count() % 2` (sol3)

### sol1 の「前半=前の行、後半=flip」との関係

これは「位置 2i は親、位置 2i+1 は親の flip」を **行のレベルで集約した結果**であり、自明ではない。背後には次の性質がある：

> 規則 `rule` と `flip` が可換: `rule(flip(c)) = flip(rule(c))`

- `flip(rule(0)) = flip(01) = 10 = rule(1) = rule(flip(0))`
- `flip(rule(1)) = flip(10) = 01 = rule(0) = rule(flip(1))`

これにより、「前の行の後半 = 前の行のflip」という性質が世代を超えて保たれる。

| 観察レベル | 得られる漸化式 | 対応する解 |
| --- | --- | --- |
| 文字単位 (位置 2i / 2i+1) | `j` の各 bit を見て flip を数える | sol3 (直接) |
| 行単位 (前半 / 後半) | `n` を 1 ずつ減らす再帰 | sol1, sol2 |

### Thue–Morse 列

この数列はそのまま **Thue–Morse 列** と呼ばれる有名な数列。
一般項が `t_n = popcount(n) mod 2` であることは Thue–Morse 列の定義的性質として知られているため、置換規則「0→01, 1→10」を見た時点で `(k-1).bit_count() % 2` と書けるパターン認識が可能。
