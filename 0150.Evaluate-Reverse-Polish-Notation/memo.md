### int(a / b) と a // b の違い

https://docs.python.org/3/library/stdtypes.html#numeric-types-int-float-complex

//: The result is always rounded towards minus infinity

どちらも整数除算だが、丸める方向が異なる。

| 式 | 丸め方向 | -7 / 2 の結果 |
|---|---|---|
| `int(a / b)` | ゼロ方向 (truncate) | `-3` |
| `a // b` | 負の無限大方向 (floor) | `-4` |

```python
int(-7 / 2)  # → -3  (-3.5 をゼロ方向に丸める)
-7 // 2      # → -4  (-3.5 を負の無限大方向に丸める)
```

両オペランドが正の場合は同じ結果になる。

### IEEE 754 の偶数丸め（banker's rounding）

https://ja.wikipedia.org/wiki/%E7%AB%AF%E6%95%B0%E5%87%A6%E7%90%86

通常の四捨五入（0.5 を常に切り上げ）とは異なり、
**ちょうど 0.5 のときは近い方の偶数に丸める**ルール。

```python
round(0.5)  # → 0  (偶数)
round(1.5)  # → 2  (偶数)
round(2.5)  # → 2  (偶数)
round(3.5)  # → 4  (偶数)
```

#### なぜ偶数丸めか

0.5 を常に切り上げると、大量の計算で誤差が一方向に蓄積する（統計的バイアス）。
偶数丸めにすることで切り上げ・切り捨てが均等になり、誤差が相殺される。
金融計算・科学計算で標準的に使われる理由がこれ。

#### Python での挙動

Python の `round()` は IEEE 754 偶数丸めに従う。

```python
round(2.5)   # → 2  (偶数丸め)
round(3.5)   # → 4  (偶数丸め)
```

ただし浮動小数点の表現誤差により、意図しない結果になることがある。

```python
round(0.1 + 0.2, 1)  # → 0.3  (OK に見えるが内部表現はズレている)
```

#### int(a / b) との関係

`int(a / b)` は `float` 経由のため、大きな整数では表現誤差が出る可能性がある。

```python
int(10**18 / 3)   # → 333333333333333344  (誤差あり)
10**18 // 3       # → 333333333333333333  (正確)
```

整数演算のみで済む場合は `//` の方が精度が高い。
ゼロ方向丸めが必要かつ精度も必要な場合は `math.trunc()` や符号を考慮した整数演算を使う。
